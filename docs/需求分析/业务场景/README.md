# 业务场景分析

本目录包含 5 个典型的**数据处理与编排**场景，用于验证 **ControlFlow** 引擎的领域模型设计。

我们关注的重点是**控制流（Control Flow）**——即如何调度、监控和管理这些任务，而非具体的数据处理逻辑。

---

## 场景概览

| 场景 | 核心控制模式 | 涉及任务类型 | 关键特性验证 |
|------|-------------|-------------|-------------|
| [场景1：批量特征工程](./场景1-批量特征工程.md) | **DAG 编排** | `PySparkOperator` | 并行执行、依赖管理、失败重试 |
| [场景2：增量数据处理](./场景2-增量数据处理.md) | **定时调度** | `SQL`, `PySparkOperator` | Cron 调度、时间窗口变量、补数 |
| [场景3：流式数据清洗](./场景3-流式数据清洗.md) | **生命周期管理** | `StreamingOperator` | 启动/停止、状态保存(Savepoint)、弹性扩缩 |
| [场景4：数据标注工作流](./场景4-数据标注工作流.md) | **事件驱动** | `Approval`, `Python` | 外部事件触发、条件分支、人工审核 |
| [场景5：用户行为分析与推荐](./场景5-用户行为分析与推荐.md) | **混合编排** | All Types | 流批混合、复杂依赖、端到端流程 |

## 核心需求总结

通过分析上述场景，ControlFlow 引擎需要满足以下核心需求：

1.  **通用 DAG 能力**: 支持串行、并行、菱形依赖等复杂拓扑。
2.  **多模态任务支持**: 统一管理 Batch（离线）、Streaming（实时）、Human（人工）等多种任务类型。
3.  **反应式控制 (Reactive Control)**: 不仅是"启动"任务，还能在运行时进行"干预"（如暂停、扩缩容、修改参数）。
4.  **事件驱动机制**: 任务状态流转依赖于内部或外部事件（Events）。
